# 당근마켓 클론 코딩 프로젝트 코드 설명서

이 문서는 `lib` 폴더 내의 코드와 사용된 라이브러리들을 플러터(Flutter) 초심자의 눈높이에서 설명합니다.

## 1. 사용된 외부 라이브러리 (pubspec.yaml)

이 프로젝트에서 사용 중인 주요 패키지(플러그인)들입니다.

| 라이브러리 이름 | 용도 | 설명 |
| --- | --- | --- |
| **get** | 상태 관리 & 라우팅 | 복잡한 `Navigator` 대신 `Get.to()`, `Get.offNamed()` 등으로 화면 이동을 쉽게 하고, 앱의 데이터 상태를 관리합니다. `main.dart`에서 `GetMaterialApp`을 사용하는 이유입니다. |
| **flutter_svg** | 이미지 처리 | SVG 벡터 이미지를 깨지지 않고 선명하게 보여주기 위해 사용합니다. (예: 아이콘들) |
| **google_fonts** | 폰트 | 앱에 다양한 구글 폰트를 쉽게 적용할 수 있게 해줍니다. |
| **shared_preferences** | 데이터 저장 | 로그인 정보나 앱 설정 같은 간단한 데이터를 기기에 영구 저장할 때 사용합니다. |
| **equatable** | 데이터 비교 | 두 객체의 값이 같은지 비교를 쉽게 도와줍니다 (주로 상태 관리 로직에서 사용). |
| **cupertino_icons** | 아이콘 | iOS 스타일의 기본 아이콘들을 사용할 수 있게 해줍니다. |

## 2. 주요 Flutter 클래스 및 위젯 설명

`login_page.dart`와 `main.dart` 등에서 자주 보이는 키워드들입니다.

### A. 화면 구성의 기본 (Widgets)

*   **`StatelessWidget`**: 상태가 변하지 않는 화면을 만들 때 씁니다. (예: `LoginPage`). 화면에 보여줄 내용이 고정되어 있거나, 단순히 보여주기만 할 때 사용합니다.
*   **`Scaffold`**: 앱의 기본적인 뼈대입니다. 흰 도화지라고 생각하면 됩니다. 보통 `body` 속성에 화면 내용을 넣습니다.
*   **`SafeArea`**: 최신 폰의 '노치'나 하단 '홈 바' 영역을 침범하지 않도록 안전한 영역 안에 콘텐츠를 표시해줍니다.
*   **`Container`**: 상자입니다. 색칠하거나, 테두리를 주거나, 크기를 정할 때 가장 많이 쓰이는 만능 위젯입니다.

### B. 배치와 레이아웃 (Layout)

*   **`Column` (열)**: 위젯들을 **세로(위→아래)**로 나열합니다.
*   **`Row` (행)**: 위젯들을 **가로(왼쪽→오른쪽)**로 나열합니다.
*   **`SizedBox`**: 빈 공간을 만들거나(간격 띄우기), 위젯의 크기를 강제로 고정할 때 씁니다.
    *   예: `SizedBox(height: 40)` → 세로로 40만큼 빈 공간을 둠.
*   **`Padding`**: 안쪽 여백을 줍니다. 내용물이 벽에 딱 붙지 않게 할 때 씁니다.
*   **`Spacer`**: 남은 빈 공간을 다 차지해버리는 녀석입니다. 위젯들을 양 끝으로 밀어버릴 때 유용합니다.

### C. 정렬 (Alignment)

*   **`MainAxisAlignment` (주축 정렬)**:
    *   `Column`일 때는 **세로** 방향 정렬을 뜻합니다.
    *   `Row`일 때는 **가로** 방향 정렬을 뜻합니다.
    *   예: `MainAxisAlignment.center` (가운데 정렬), `MainAxisAlignment.spaceBetween` (양 끝으로 벌리기).
*   **`CrossAxisAlignment` (교차축 정렬)**:
    *   `Column`일 때는 **가로** 방향 정렬 (뚱뚱해지는 방향).
    *   `Row`일 때는 **세로** 방향 정렬.

### D. 디자인과 꾸미기 (Style)

*   **`TextStyle`**: 글자 크기(`fontSize`), 굵기(`fontWeight`), 색상(`color`) 등을 지정합니다.
*   **`BoxDecoration`**: `Container`를 꾸밀 때 씁니다. 배경색, 테두리(`border`), 둥근 모서리(`borderRadius`) 등을 설정합니다.

### E. GetX 관련 (이 프로젝트의 특징)

*   **`GetMaterialApp`**: `main.dart`에 있으며, GetX 라이브러리 기능을 쓰기 위해 기본 `MaterialApp` 대신 사용한 것입니다.
*   **`Get.offNamed('/home')`**: 화면을 이동하는 코드입니다. `offNamed`는 "이전 화면 기록을 지우고 이동한다"는 뜻으로, 로그인 후 뒤로가기를 눌러도 다시 로그인 화면으로 오지 않게 할 때 씁니다.

## 3. 프로젝트 구조 요약

*   **`lib/main.dart`**: 앱의 시작점입니다. 여기서 `GetPage` 목록을 통해 앱의 지도(라우팅)를 정의하고 있습니다.
*   **`lib/src/pages/`**: 실제 화면들이 모여있는 곳입니다. (`login`, `home` 등)
*   **`lib/src/common/`**: 여러 화면에서 공통으로 쓰이는 부품(컴포넌트)들이 들어갑니다.
